# Fork Differences

This document summarizes the changes in this fork compared with `origin/main`.

## Functional Differences

### 1. WhatsApp can forward self-sent messages

- You can trigger nanobot from the same WhatsApp account that is running the bridge.
- Echo loops are handled on the Python side by prefix-based filtering.

### 2. Bot replies are prefixed for loop prevention

This fork prefixes outbound WhatsApp bot messages with `🤖 `.

- Messages generated by nanobot are visually distinct in the chat.
- The inbound handler can skip echoed bot messages that come back from WhatsApp.

### 3. `/no` messages are ignored on WhatsApp

- You can send local control or note-like messages without triggering the bot.
- This is separate from the bot-prefix filtering and applies to user input.

### 4. Progress streaming is disabled by default

- Channels do not receive intermediate progress text unless explicitly enabled.
- WhatsApp specifically skips `_progress` messages before sending.
- Chat output is cleaner and avoids noisy partial updates.

## Development And Deployment Differences

### 5. The WhatsApp bridge auto-rebuilds when sources change

- Local bridge changes are picked up without manually deleting `~/.nanobot/bridge`.
- Development flow is more reliable when iterating on the TypeScript bridge.
